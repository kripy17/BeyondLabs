BeyondArch — High-Impact Improvements (No Filler)

---

🔴 Tier 0 — Things that are actually broken right now

1. Backend URL is client-only + no health signal in UI
`src/lib/backend.ts` reads `localStorage` on every call. There's no visible "backend offline" state anywhere in the shell — users hit Nmap, wait 150s, then see an ErrorBanner. 
Fix: A persistent pill in the topbar that pings `/api/hackingtool/categories` every 30s and shows `● online 42ms` / `● offline`. When offline, disable every "Run" button globally via a context, don't let the request fire at all.

2. `postJson` / `uploadFile` have no AbortController
Every long scan (nmap 150s, maigret 180s, harvester 140s) is uncancellable. If a user navigates away, the fetch keeps running and setState-after-unmount warnings flood the console.
Fix: Every API function accepts `{ signal }`. Every page uses `useEffect` cleanup + a "Cancel" button next to the loading panel.

3. `dangerouslySetInnerHTML` / raw rendering of backend stdout
`TerminalOutput` and `CodeBlock` render `stdout` from executed binaries. If backend is compromised or a tool outputs crafted ANSI/HTML, you have XSS. Even if you strip tags, ANSI escape codes render as garbage.
Fix: Run stdout through a real ANSI-to-HTML parser (`ansi-to-html` package) with a strict allowlist, then render as text nodes — never `dangerouslySetInnerHTML` on untrusted output.

4. `localStorage` swallow-all `catch {}` in `prefs.tsx`, `theme.tsx`, `backend.ts`, `recents.ts`
Quota exceeded → silent data loss. Corrupt JSON → app boots into default state and user loses their case history.
Fix: One `safeStorage` helper that logs to a visible toast when quota/parse fails. Case data + investigation notes should go to IndexedDB (via `idb-keyval`), not localStorage.

5. Theme FOUC on first paint
Theme is applied in a `useEffect` inside `ThemeProvider`. SSR/first render always flashes the default theme before hydration swaps it.
Fix: Inline `<script>` in `__root.tsx` head that reads `localStorage` and sets `data-theme` on `<html>` before React mounts. This is the standard next-themes pattern.

---

🟠 Tier 1 — Structural problems limiting the whole app

6. Every page is an island — no cross-tool handoff actually works
You have `handoff.ts` but I don't see any page consuming it meaningfully. A user extracts IOCs on the Parser page, then has to copy-paste an IP to the Recon page, then copy-paste to Reputation. That's 3 tools, 6 clicks.
Fix: Every IOC in any output is a `<IocChip>` with a right-click context menu: "→ Reputation", "→ Recon", "→ VirusTotal", "→ Add to case". One shared component, wired to router navigation with pre-filled state via search params (`?target=1.2.3.4`).

7. No case as first-class citizen
Case & Report exists, but nothing else knows about it. Every page should have a "📎 Attach to case: [Active Case Name ▼]" in the top-right of the output section. Every action logs an evidence entry (timestamp, tool, input, output hash) to the active case automatically.
Fix: `useActiveCase()` hook. `<AttachButton payload={result} />` primitive on every OutputSection. Case timeline becomes a real forensic log, not a manual markdown writeup.

8. Search bar in topbar and ⌘K palette are redundant
Two search UIs doing similar things. Pick one. The ⌘K palette is the correct primitive — kill the topbar search, use that space for the backend health pill + active case indicator + user menu.

9. Sidebar has 18 items across 6 groups with no visual hierarchy
Everything is the same weight. Users don't know what's "primary" (Parser, Case) vs "utility" (URL codec, Timestamp). 
Fix: Two-tier sidebar. Top: Investigate (Parser, Case, Terminal, Command Deck) — always visible, larger. Bottom: collapsible Tools and Utilities groups. Add pinned favorites.

10. Results are ephemeral
Every result is thrown away on navigation. A user runs a 3-minute nmap scan, clicks to Recon to look something up, comes back — result is gone.
Fix: Per-workspace result cache in Zustand or React Query with `staleTime: Infinity`. Show "Last run: 2m ago" chip. "Re-run" and "View history" (last 5).

🟡 Tier 2 — Visual & UX changes that will actually be felt

11. Typography is inconsistent despite the `ba-title/ba-h2/ba-label` scale
I've written it into `styles.css` but audit shows pages still use raw `text-mono text-[11px]` inline classes. Do a pass replacing every ad-hoc size with the scale token. This is the single biggest reason the app "feels like a mix of old CSS."

12. Output sections are visually flat — no scannable hierarchy
Every OutputSection looks the same weight. A "Verdict: MALICIOUS" banner should dominate; a "Raw headers" section should be collapsed by default.
Fix: Introduce section priorities: `primary` (verdict, IOCs) always expanded and 2x visual weight, `secondary` (metrics, timeline) expanded, `raw` (stdout, headers, JSON) collapsed with a summary chip ("142 lines, 3.4 KB").

13. No skeleton shapes match the real output shape
Current `Skeleton` is generic gray lines. When results load they pop in and shift layout. 
Fix: Per-page skeletons that match the actual final layout — nmap should skeleton a port table, phishing should skeleton the verdict banner + auth chips.

14. Empty states are missing personality
The `Empty` component exists but most pages just show "no results." Empty states should show what a good result looks like — a faded example screenshot / mini-diagram — so users know what they'll get.

15. No motion — the app feels dead
You have framer-motion in the stack (I assume). Zero use of it. Add: (a) OutputSection expand/collapse with height animation, (b) new IOCs animate in with a stagger, (c) verdict banner has a subtle pulse on first render, (d) command palette has a proper spring open.

16. Themes still feel like recolors
Broadsheet/Late Edition were the exception. Terminal Noir, SOC Console, Aurora, Solar — these differ in color but share the same shadows, radii, border weights, animation timing. 
Fix: Each theme owns its own `--radius`, `--border-width`, `--shadow-*`, `--font-display`, `--transition-timing`. Terminal Noir = 0 radius, hard shadows, no transitions (instant/hard). Aurora = 12px radius, soft glows, 300ms ease-out. Then themes actually feel different.

17. No dark/light toggle within a theme
Editorial Dark and Late Edition are separate themes. That's wrong — theme = personality, mode = light/dark. 4 themes × 2 modes = 8 combos with one switcher for each axis.

🟢 Tier 3 — Features that make it a product, not a demo

18. Investigation timeline (single killer feature)
A `/timeline` view that shows every action taken in the current case: "14:02 Extracted IOCs from email → 14:03 Enriched 1.2.3.4 (VT: 12/89 malicious) → 14:05 Ran nmap → …". Exportable as PDF. This is what analysts actually need and no tool does well.

19. Diff view for reruns
Ran nmap yesterday and today? Show diff: 2 new ports open, 1 port closed. Same for VT reputation ("+3 detections since last check"). Massive value for monitoring.

20. Keyboard-first everything
- `j/k` navigate results
- `y` copy row
- `e` enrich selected IOC  
- `c` attach to case
- `.` context menu on selection
Show `?` overlay listing them.

21. One backend, one Docker command
Ship a `docker-compose.yml` at the repo root: web + FastAPI backend + all the CLI tools baked in. Right now the barrier to trying this is "clone two repos, install Python, install nmap, install…". Kill that.

22. Server-Sent Events for long scans
Nmap runs 150s with a spinner. Stream stdout line-by-line via SSE from the FastAPI backend so users see progress as it happens. Feels 10x more responsive without being faster.

23. Report generator that isn't just markdown
Current Case export = markdown. Add: styled HTML report with theme, embedded charts (port scan visual, MITRE heatmap), one-click "Export as PDF" via browser print stylesheet. Analysts hand these to managers.

More High-Impact Improvements — Round 2

Continuing from the last list, going deeper into things I've actually seen in the codebase.

---

🔴 Data & State Layer

24. No single source of truth for API state
Every page does `useState` + `try/catch` + manual loading flags. That's why loading/error handling was inconsistent across 18 pages in the first place.
Fix: TanStack Query for every backend call. `useQuery` for reads, `useMutation` for actions. You get caching, retries, dedup, background refetch, `isPending`/`isError` for free. Kills 300+ lines of boilerplate.

25. `apiClient.js` is JavaScript, not typed
Every API response is `any`. TypeScript can't help you when the backend contract drifts. I've seen `result.events || []`, `result.detections || []` — defensive because you don't know the shape.
Fix: Convert to `.ts`. Define Zod schemas for every response. `postJson` becomes `postJson<T>(path, body, schema)` and parses with `schema.parse(data)` — you catch backend drift at the boundary, not 3 components deep.

26. No request deduplication
Two components on the same page both call `getHackingtoolCategories()` on mount → 2 network requests. Happens everywhere.
Fix: Falls out of #24 free.

27. Case data lives only in one browser
Zero export/import for cases. Zero sync. Analyst switches laptops → gone.
Fix: JSON export/import (button pair), then optional Lovable Cloud sync later.

---

🟠 Component Architecture

28. `Workspace.tsx` is a god component
It has Empty, Skeleton, ErrorBanner, OutputSection, CodeBlock, MetricTile all crammed in. Hard to find, hard to change one without side effects.
Fix: Split into `src/components/output/` folder — one file per primitive, one barrel export. Same for `soc/` folder.

29. Prop drilling for prefs/theme/backend URL
`usePrefs()` called in 30+ components. Every re-render of the provider re-renders the whole tree.
Fix: Split contexts by concern (`useThemeMode`, `useDensity`, `useFonts`) so components subscribe to exactly what they need. Or move to Zustand with selectors.

30. `CommandPalette` re-computes items on every open
`GROUPS.map(...).items.map(...)` runs every render. Fine now, painful at 50+ tools.
Fix: `useMemo` + fuzzy search with `cmdk`'s built-in ranking or `fuse.js`.

31. Icons imported individually across the app
Every route imports 5-10 lucide icons. Bundle bloat.
Fix: Centralize in `src/lib/icons.ts` re-exports, or use lucide's tree-shakeable `lucide-react/dynamicIconImports` for rarely-used ones.

---

🟡 UX / Interaction

32. No undo anywhere
Delete a case? Gone. Clear terminal history? Gone. Reset preferences? Gone.
Fix: Every destructive action → toast with 5s undo. Sonner supports this out of the box.

33. Copy-to-clipboard has no feedback
Users click "copy" and don't know it worked. I've written `CodeBlock` with a copy button but it doesn't animate/toast.
Fix: Icon swap to checkmark for 1.5s + subtle scale animation. Standard pattern.

34. Forms have no validation feedback until submit
Nmap accepts any target string, sends to backend, backend errors. User waits 3s to learn they typed "htp://example".
Fix: Zod validation on the input with inline error under the field. Disable submit until valid.

35. No confirm for expensive/destructive ops
"Run full port scan" on a random IP fires immediately. No "are you sure, this hits the network for ~2 min" gate.
Fix: Destructive/expensive buttons open a confirm sheet showing: target, estimated time, what it does, "I have authorization" checkbox.

36. Terminal has no split panes or tabs
Real analysts run 3-4 things at once.
Fix: Tabs at the top of `/terminal`. Optional split (horizontal/vertical) via `react-resizable-panels`.

37. No global "recently viewed"
`recents.ts` exists but I don't see it surfaced anywhere prominent. Should be a strip below the topbar showing last 5 IOCs/targets you touched.

38. Search results have no keyboard preview
Highlight a search result → show preview panel to the right. Notion/Linear do this.

---

🟢 Visual Polish (Real, Not Generic)

39. Hairline dividers are inconsistent
`border-border`, `border-border/60`, `border-border/40` all used arbitrarily. Density feels random.
Fix: Two divider tokens: `--divider-strong` (section boundaries), `--divider-soft` (within a section). Nothing else.

40. No focus-visible styles
Tab through the app — you can't see where you are. Accessibility failure and looks unpolished.
Fix: Global `:focus-visible` ring using theme accent color, 2px offset, per-theme radius.

41. Data tables are just `<div>` grids
No sorting, no resize columns, no sticky headers, no row hover. Nmap port list, SIEM events, OSINT results all need real tables.
Fix: TanStack Table + a shared `<DataTable>` component. Once.

42. No syntax highlighting in CodeBlock
Raw stdout looks like text. JSON should highlight, headers should highlight, nmap output has known structure.
Fix: `shiki` (build-time themes match your app themes) or `prism-react-renderer`. Language detection based on content.

43. Icons and text baseline don't align
Look at any sidebar item — icon center is not on the text cap-height line. Universal small-app tell.
Fix: `inline-flex items-center` + adjust icon `stroke-width` and translate-y by 0.5px. Do it once in `SidebarMenuButton`.

44. No loading progress for known-slow ops
Nmap says 150s max. Show an indeterminate progress bar for the first 5s, then a determinate one estimating based on typical duration.

45. Toast placement collides with sidebar trigger on mobile
Sonner defaults to bottom-right. On mobile, sidebar trigger and toasts stack. Move toasts to top-center on mobile.

46. Cards have no elevation hierarchy
Every Panel is the same shadow depth. Verdict panels should float higher.
Fix: 3 elevation tokens per theme: `--elevation-base`, `--elevation-raised`, `--elevation-floating`.

---

🟣 Backend Contract & Reliability

47. No retry policy for transient network errors
One dropped request → full ErrorBanner. Should retry 3× with exponential backoff for 5xx/timeout.
Fix: TanStack Query `retry: 3, retryDelay: exponential` (#24 again).

48. No request-level timeout — only backend-level
`postJson` accepts `timeout` param but not used everywhere. Some calls hang until browser gives up.
Fix: Every call has a default 15s timeout, expensive ones set explicitly.

49. No rate limit protection
User spams "Run" → 10 nmap scans queue. Backend melts.
Fix: Disable button while `isPending`. Optional: per-tool cooldown pill ("cooldown 3s").

50. No CSRF/auth story
Backend is open. Anyone on the same LAN can hit it.
Fix: Even for local: shared secret in localStorage → `X-BeyondArch-Key` header, backend rejects without it. Displayed once on first setup.

🔵 Content & Copy

51. Empty states say "no results" — useless
Should say: what to do next, and why this tool exists. "Paste an email header to see SPF/DKIM/DMARC verdicts and pull IOCs."

52. Errors are technical
"backend 500 Internal Server Error" — user has no path forward.
Fix: Error mapping table: known errors → human message + suggested action ("Backend can't reach nmap binary — check `nmap --version` on the backend host").

53. Tool descriptions in the sidebar are one-liners with no examples
Add hover tooltips with a 1-line example of input.

54. No changelog / "what's new"
Add a small `NEW` pill next to Terminal for 14 days after release. Modal on first launch showing the last 3 shipped features.

⚫ Meta / Repo

55. `.js` and `.tsx` mixed in `src/api/`
Everything else is TS. This is legacy debt. Migrate — it's ~10 small files.

56. No Storybook / component gallery
You've built 20+ shared primitives. No way to see them all in isolation to catch drift.
Fix: Ladle (lighter than Storybook) or a `/dev/components` route gated by env.

57. No e2e tests for critical flows
"Extract IOC → send to Recon → attach to case" is the app's whole value prop. One Playwright test guarding it prevents regressions forever.

58. No performance budget
Bundle size is likely huge (framer-motion + shadcn full + lucide + shiki). Add `bundle-visualizer` check to CI, budget of 300kb initial JS.

59. No error tracking
When the app crashes for a user, you never know. Add `error-capture.ts` (already exists) → wire to Sentry or a self-hosted GlitchTip.

60. README doesn't sell it
Screenshots are half the sell of a security tool.
